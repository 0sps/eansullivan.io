Program description for developers

Assumptions Made Before Coding Tecmo Bowl
- User will follow all instructions in manual and How to Play button
- User is capable of operating Proteus and using fine motor skills
- User knows basic football concepts such as the “endzone” and scoring a “touchdown”
- The user is not trying to break game; user is trying to play game for entertainment

 
Key Variables
- Player user, comp;
- These two Player objects are the players in the game and the two main variables in the game.
- Each Player object has four key variables that allow them to be moved around the screen:
int x, y, hX, hY;
The x and y variables give the coordinates of the top left point of
the body of the player and hX and hY give the coordinate of the top left
point if the helmet of each player. These four variables are what are
manipulated every time the player moves to create motion on the screen.
Each Player object also has a variable int position that is a 1 if it is the user and a 2 if it is the AI. This allows for the proper coloring of the players’ helmets and jerseys.
The variable bool useRight is also used in the program to
determine whether the AI is moving to the left or right. When it is
false, the defender moves to the left. It becomes true just as the
player would hit the left wall so that he then slides to the right. It
goes back to false just as he would hit right wall, and so on.
 
Functions
Static Methods
void setField()
This method creates the field background and the directional arrows
used in gameplay. Method is called when the game is launched from Start
menu:
if(x>=80 && x<= 160 && y>=80 && y<=130)
setField();
bool isTackled(Player a, Player b)
This method determines if Player a is undergoing a head-on collision
with Player b, in which he would be “tackled.” Returns true if there is a
matching x and y coordinate on the front of both players, meaning they
are colliding:
while(!isTackled(user, comp)){
}
Player Class Methods
Player(const char t[], int p)
The constructor asks for the team name of the player and the integer that determines user or AI:
Player user= Player(“OSU”, 1);
void drawPlayer(int p)
Method asks for position of player to determine colors and then draws
a rectangle for the body and a square for a helmet based on the
object’s x,y,hX, and hY values:
user.drawPlayer(1);
void moveUp()
The method decrements the hY and y values and redraws the player so he is translated up three pixels:
user.moveUp();
void moveDown()
The method increments the hY and y values and redraws the player so he is translated down three pixels:
user.moveDown();
void moveLeft()
The method decrements the hX and x values and redraws the player so he is translated left three pixels:
user.moveLeft();
void moveRight()
The method increments the hX and x values and redraws the player so he is translated right three pixels:
user.moveRight();
int getX()
Method returns Player’s x:
user.getX();
int getY()
Method returns Player’s y:
user.getY();
int getHX()
Method returns Player’s hX:
user.getHX();
int getHY()
Method returns Player’s hY:
user.getHY();
bool inEndZone()
Method returns true if the Player’s y value is equal to or less than
the bottom y value of the endzone on screen, and false if he is not yet
there:
if(user.inEndZone()){
}
 
Program Performance and Limitations
The program is able to have motion on the screen without excessive
flickering, which allows for more crisp gameplay. The game is limited in
the intelligence of the artificial intelligence and in controlling the
speed of the artificial intelligence’s motion relative to that of the
player. This allows for the game to be relatively easy for the user to
score a touchdown, which is a potential plus for the company because
initial users will enjoy the success and want to continue to play. The
program performance allows for a Start menu with options that always
reverts back to the main menu, giving the user the capability to utilize
all of the options that the Tecmo Bowl game has to offer.
